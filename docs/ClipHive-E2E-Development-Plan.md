## 0. Multi-Agent Execution Guide

> **For Claude Code agents:** This section is the orchestration contract for building ClipHive in one day using parallel subagents. Read this entire section before touching any code. Every agent must read its own role card and the shared contracts file before starting work.

### 0.1 Prerequisites Before Spawning Any Agent

The **Orchestrator** (the human or a root Claude Code session) must complete these steps first — they take ~30 minutes and unblock all parallel streams:

**Step 1 — Bootstrap repo:**
```bash
git clone https://github.com/<org>/ClipHive.git && cd ClipHive
dotnet new sln -n ClipHive
mkdir -p src/ClipHive tests/ClipHive.Tests installer docs assets/icon
```

**Step 2 — Write shared contracts file** (`src/ClipHive/Contracts.cs`).  
All agents code against these interfaces. Do NOT start parallel agents until this file exists.

```csharp
// src/ClipHive/Contracts.cs
// AUTO-GENERATED BY ORCHESTRATOR — DO NOT EDIT IN FEATURE BRANCHES
// All agents depend on this file. Changes require orchestrator approval.

namespace ClipHive;

public interface IStorageService
{
    Task AddAsync(string plaintext, string? sourceApp = null);
    Task<IReadOnlyList<ClipboardItem>> GetAllAsync();
    Task<IReadOnlyList<ClipboardItem>> SearchAsync(string query);
    Task DeleteAsync(long id);
    Task DeleteAllAsync(bool keepPinned = true);
    Task DeleteOlderThanAsync(DateTime cutoff, bool keepPinned = true);
    Task SetPinnedAsync(long id, bool pinned);
}

public interface IEncryptionHelper
{
    (string Ciphertext, string Iv, string Tag) Encrypt(string plaintext);
    string Decrypt(string ciphertext, string iv, string tag);
}

public interface ISettingsService
{
    AppSettings Load();
    void Save(AppSettings settings);
}

public interface IHotkeyService
{
    event EventHandler? HotkeyPressed;
    bool Register(IntPtr hwnd, uint modifiers, uint virtualKey);
    void Unregister(IntPtr hwnd);
}

public interface IPasteService
{
    bool IsPasting { get; }
    Task PasteAsync(string content);
}

public interface IClipboardMonitorService : IDisposable
{
    event EventHandler<string>? ClipboardChanged;
    void Start();
    void Stop();
}

public interface IAutoClearService : IDisposable
{
    void Start();
    void Stop();
}
```

All agents must follow these rules to prevent git merge conflicts:

| Rule | Detail |
|------|--------|
| **File ownership** | Each agent owns specific files. No agent edits another agent's files. |

## 1. Project Overview

**ClipHive** is a lightweight, privacy-first Windows clipboard history manager inspired by macOS's Maccy. It lives in the system tray, monitors clipboard changes, stores encrypted history locally, and surfaces a fast keyboard-driven sidebar when you need to paste something you copied earlier.

### Goals

- Zero network calls — no telemetry, no cloud sync
- All data AES-256-GCM encrypted at rest
- <10 MB RAM footprint at idle
- Sub-50 ms sidebar popup latency
- Professional open-source project: clean repo, CI, installer, docs

### Non-Goals (v1.0)

- Image or file clipboard capture (text only for v1)
- Cross-device sync
- Plugin system

## 2. Architecture Decision Record

### ADR-001: WPF over WinForms or Electron
**Decision:** WPF (.NET 8)  
**Rationale:** Native Windows rendering with GPU compositing; MVVM support for testable UI logic; no Node.js/Electron overhead. WinForms lacks the animation/overlay capabilities needed for a Maccy-style sidebar.

### ADR-002: SQLite over flat files or LiteDB
**Decision:** Microsoft.Data.Sqlite  
**Rationale:** Structured queries for filtering/sorting history; WAL mode for non-blocking writes; well-understood disaster recovery via file copy.

### ADR-003: AES-256-GCM for encryption
**Decision:** `System.Security.Cryptography.AesGcm`  
**Rationale:** Authenticated encryption prevents tampering; random 12-byte IV per record ensures ciphertext uniqueness; .NET 8 built-in, no extra dependencies.

### ADR-004: Win32 P/Invoke over third-party hook libraries
**Decision:** Raw P/Invoke for `RegisterHotKey`, `WM_CLIPBOARDUPDATE`, `SendInput`  
**Rationale:** No NuGet package to go stale; full control over message loop; smaller dependency surface.

### ADR-005: Inno Setup 6 over MSIX/WiX
**Decision:** Inno Setup 6  
**Rationale:** Smallest installer size; well-documented; no Windows Store signing requirement; single Pascal script easy to review in open source context.

### Phase 2 — EncryptionHelper

**Goal:** Deterministic AES-256-GCM encryption/decryption with random IV per operation.

**File:** `Services/EncryptionHelper.cs`

```csharp

    // Encrypt returns (ciphertext, iv, tag) — all base64
```

- Branch protection on `main`: require 1 review, require CI to pass, no force push
- Branch protection on `develop`: require CI to pass
- Enable Discussions for feature requests
- Enable Security Advisories
- Add topics: `clipboard`, `windows`, `productivity`, `wpf`, `csharp`, `privacy`
